# Personal Claude Code Rules

- Start by understanding the repository before editing files.
- Prefer the smallest safe change that solves the problem.
- State assumptions clearly and verify them from the codebase.
- Prefer official docs and repository files over guessing.
- After code changes, suggest the exact validation steps to run.
- Do not touch secrets, .env files, or production credentials unless explicitly asked.
- For major refactors, present a short plan first.
